I build systems — not features, not demos.

My work moves in one direction: from fragmented, human-dependent processes toward structured, automated, and scalable infrastructure. This applies whether I'm writing backend architecture, designing AI pipelines, integrating hardware, or mapping how financial systems should evolve.

I operate on two layers simultaneously. The execution layer is where I ship — real systems with real constraints. The infrastructure layer is where I think about how systems should be structured at scale, before anyone else needs to care about it.

That being said, having agents to help does not mean they can be relied on. Humans are capable of reasoning about the unknown, while LLMs are constrained by prior data and learned weights. That distinction matters, which is why human involvement remains essential.

---

## Work

**TUAMS**

TUAMS is a system-level construct, not a standalone app. It is built with extremely detailed principles: structured coordination, intelligent behavior at the component level, and extensibility by design. The goal is a framework where organization and automation are architectural — not bolted on. This is the project I treat most seriously, because it is the one most directly about how I think systems should work.

**"Infrastructure for a New World"**

A research paper, not a manifesto. It analyzes why current financial and data systems are structurally inefficient — fragmented, manually dependent, and resistant to scale — and lays out the technical and architectural conditions that make automated, globally accessible infrastructure viable. The argument is analytical. The conclusions are system-focused.

**SAT Schedule Creator**

An end-to-end intelligent pipeline: OCR extraction of structured data, LLM-driven output generation, and a clean frontend layer. Built to solve a real problem without overengineering the interface. The architecture matters more than the surface.

**Gesture-Controlled Drone**

Real-time gesture recognition using computer vision, wired into hardware — ESC, flight controller, dynamic control logic. This project exists because I wanted to work with systems that don't have a retry button. Latency and precision are not optional in hardware.

**Investing Playbook**

A structured framework for market analysis. Not stock picks — a decision system built around risk modeling, narrative decomposition, and structured reasoning under uncertainty. Markets are mostly efficient. Edge lives in interpretation and execution.

## Thinking

Systems-first. I care about interactions and structure before I care about individual components. Clarity over complexity. Real-world applicability over theoretical elegance. Structure that can survive contact with scale.

On markets: edge does not come from information — it comes from how you frame and act on it. I build the same way I invest: with a defined thesis, clear assumptions, and explicit risk.
